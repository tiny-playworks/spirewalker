# Act1 -> Act2 渗透率修复方案（最终执行版）

## 摘要

- 本轮目标只剩一个：**提高 Act1 -> Act2 渗透率**，让 Act2 验证段拿到足够样本。
- 不碰 Act2 验证段实现，不碰 Act2 白名单、地图、状态继承、runner 核心逻辑。
- 执行闭环固定为：
  1. 先做 Boss 失败归因摘要
  2. 明确唯一主拦截 Boss
  3. 只对该 Boss 做一次单变量修改
  4. 提供修改前后 Boss 定义 diff
  5. 自动校验仅声明字段发生变化
  6. 跑多 seed 聚合验证
  7. 根据结果决定保留、冻结或回滚

## 目标与阶段门槛

- 硬门槛：`Act1 boss defeat rate >= 10%`
- 理想门槛：`Act2 entry sample count >= 30`
- 解释规则：
  - 未达硬门槛：只说明 Act1 渗透未打通
  - 达到硬门槛但 `Act2 entry sample count < 30`：仍不能严肃解读 Act2 encounter 压力
  - 达到理想门槛：可结束阶段 1，并恢复 Act2 压力验证为主要任务
- `mixed` 本轮只作为入口渗透观测对象，不是优化目标。

## 阶段 0：Boss 失败归因摘要

- 修改前必须先输出只读归因摘要。
- 必须输出：
  - `slime_boss` 击杀占比
  - `act1_boss_gate` 击杀占比
  - 两者死亡 phase 分布
  - 平均死亡前玩家 `hp / block`
  - 平均死亡时敌方剩余血量
- 主拦截 Boss 判定规则：
  1. 若击杀占比差距 `>= 10%`，直接判定主拦截 Boss
  2. 仅当差距 `< 10%` 时，才继续看：
     - 死亡窗口是否更集中
     - 死亡时敌方剩余血量是否更低
- 输出后必须明确写出：
  - `当前主拦截 Boss = XXX`
- 阶段 1 只允许修改这个唯一 Boss，禁止同时动两个 Boss。

## 阶段 1：Boss 单变量调参

### 单变量规则

- 一轮只允许改 **一个 Boss**
- 一轮只允许改 **一个旋钮**
- 单变量旋钮白名单仅限：
  - 技能数值
  - phase 阈值
  - rotation 顺序
- 明确禁止修改：
  - 技能类型
  - AI 行为逻辑
  - 机制定义
- 不允许一次同时调整：
  - 数值 + 阈值
  - 数值 + 顺序
  - 阈值 + 顺序
- 不允许同轮同时改两个 Boss

### 修改说明与校验

- 本轮所有修改必须在“修改说明”中显式列出。
- 修改说明必须包含：
  - 目标 Boss
  - 唯一旋钮类型
  - 具体字段
  - 修改前值
  - 修改后值
  - 明确声明“未声明字段不变”
- 必须提供 Boss 定义修改前后 diff。
- 必须自动校验：
  - 只有声明字段发生变化
  - 未声明字段无改动
- 若校验失败：
  - 本轮视为无效
  - 结果不得用于继续调参

### 每轮固定流程

1. 输出 Boss 失败归因摘要
2. 明确：
   - `当前主拦截 Boss = XXX`
3. 输出本轮修改说明
4. 输出 Boss 定义前后 diff
5. 跑自动校验：
   - 仅声明字段发生变化
6. 跑多 seed 聚合验证：
   - `Act1 boss reach rate`
   - `Act1 boss defeat rate`
   - `Act2 entry sample count`
   - `Act2 floor 1-3 survive sample count`
   - `Act2 floor 1-5 survive sample count`
7. 重新输出 Boss 失败归因摘要
8. 根据冻结 / 回滚规则决定去留

## 回滚与冻结机制

### 回滚条件

- 若本轮后出现以下任一情况，必须回滚本轮修改，不得叠加继续调参：
  - `Act1 boss defeat rate` 下降
  - `Act2 entry sample count` 未提升，且主拦截点未变化
  - 修改校验失败
- 回滚后本轮作废，下一轮重新从阶段 0 开始。

### 成功版本冻结

- 若当前版本达到本阶段目标且表现最优，则该版本冻结为新基线。
- 冻结后不允许在该版本上叠加未验证修改。
- “表现最优”按以下顺序判断：
  1. 是否达到当前阶段目标
  2. `Act2 entry sample count` 是否更高
  3. `Act1 boss defeat rate` 是否更高
  4. 若仍接近，则优先选择改动更小的版本

## 阶段 1 结束条件

### 常规结束条件

- 同时满足以下条件，才允许进入阶段 2：
  - 当前主拦截 Boss 击杀占比 `< 40%`
  - 且另一 Boss 成为主要击杀来源

### 提前终止条件

- 若 `Act2 entry sample count >= 30`，可提前结束阶段 1。
- 这条优先级高于“主拦截 Boss 击杀占比仍 >= 40%”。
- 也就是说：
  - 只要入口样本已经足够支撑 Act2 验证，就不再继续围绕 Boss 做额外调参。

## 阶段 2：仅在必要时做 Act1 中后段供给修复

- 只有在未触发阶段 1 提前终止，且满足阶段 1 常规结束条件后，才允许进入阶段 2。
- 本阶段只允许做最小供给修复：
  - 调整 Act1 中后段 reward / shop 倾向，让已有恢复型、bridge 型、节奏修复型内容更容易出现
- 明确禁止：
  - 新增 mixed 专项 build 支持
  - 扩写第三路线体系
  - 回头改首精英
  - 动 Act2 内容
- mixed 防误调规则：
  - 本轮不得以“提升 mixed 表现”为目标做任何调整
  - 包括但不限于：
    - 卡池权重定向倾斜
    - reward 倾向定向倾斜
    - shop 出现率定向倾斜

## 测试与验收

- 必须持续通过：
  - 现有 Act1 首精英测试
  - 现有 Act2 验证段测试
  - 现有多 seed runner 测试
- 需要新增或更新的验证：
  - Boss 失败归因摘要输出稳定
  - Boss 定义前后 diff 可稳定生成
  - 自动校验能确认仅声明字段变化
  - 回滚条件触发时不会叠加继续调参
  - 冻结版本可作为新基线继续比较
- 结果解读顺序固定：
  1. 先看唯一主拦截 Boss 是否明确
  2. 再看单变量是否提高 `Act1 boss defeat rate`
  3. 再看 `Act2 entry sample count` 是否提升
  4. 达到理想门槛前，不输出 Act2 encounter 压力结论

## 假设

- 默认下一轮仍从阶段 0 开始，不直接修改 Boss。
- 默认“未声明字段不变”是硬约束，不是建议。
- 默认 mixed 不拥有独立调优目标，本轮任何 mixed 定向优化都视为越界。
- 默认一旦出现达到目标且表现最优的版本，即冻结为新基线，不继续叠加未验证修改。
