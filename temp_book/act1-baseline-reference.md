# Act1 基线状态参考（供 Act2/Act3 延续）

## 文档目的

沉淀当前 Act1 平衡状态与验证结论，作为后续 Act2/Act3 设计与调参的对照基线。  
本文件只记录“事实与经验”，不引入新的平衡改动。

---

## 一、当前 Act1 基线（已确认保留）

### 1) 基线结论

- 当前 Act1 首精英池具备筛选力，不是“白给”。
- `walker-burst` 处于健康区间，稳定成为首精英通过率最高路线。
- `walker-guard` 已脱离早期的 0% 困境，但仍低于 burst。
- `walker-mixed` 依旧是高风险路线，样本与非战斗终止率需持续观察。

### 2) 多 seed 稳定性验证（runs=50, seeds=1001~1005）

命令：

```bash
pnpm simulate:act1 -- --runs 50 --seeds 1001,1002,1003,1004,1005
```

聚合结果：

- 防守流 `firstEliteWin`：平均 `29.4%`，min `29.4%`，max `29.4%`
- 爆发流 `firstEliteWin`：平均 `62.4%`，min `61.3%`，max `63.3%`
- 混合流 `firstEliteWin`：平均 `11.9%`，min `11.1%`，max `12.5%`
- 混合流首精英样本平均：`8.40`
- 混合流 `non-battle` 平均占比：`23.6%`

按路线拆分的首精英怪物通过率均值：

- 防守流：执行者 `28.6%`，双刃猎手 `28.6%`，收债修士 `33.3%`
- 爆发流：执行者 `79.1%`，双刃猎手 `29.5%`，收债修士 `67.3%`
- 混合流：执行者 `0.0%`，双刃猎手 `43.3%`，收债修士 `0.0%`

## 二、Act1 已验证的关键实验结论（用于避免重复试错）

### 1) 首精英单变量实验结论

- 对 `act1_twin_hunter`：
  - 调 `punishMulti`（伤害/门槛）收益不明显。
  - 调基础多段输出（`multi`）有可观测收益。
  - 结论：其主要压力更接近“基础输出”，而非“惩罚触发机制”。

- 对 `act1_executioner`：
  - 单独下调 `heavy`、单独去掉 `vulnerable`、单独改 `counter` 阈值都曾出现低敏感/无效样本。
  - 轮转顺序调整（`counter -> attackBuff -> heavy`）验证有效且副作用可控。
  - 结论：executioner 压力更偏“节奏链条”而非单个数值点。

- 对 `act1_debt_monk`：
  - 在某些阶段是 guard 的关键阻断源之一，但并非所有阶段的第一主杀手。
  - 结论：其威胁是“阶段性主杀手”，需结合按怪拆分数据判断。

### 2) Guard 专项牌方向结论

- `guard_strike` 方向正确，但 early 暴露方式容易造成系统扰动。
- `anchor_slash` 被验证为有效方案，且已纳入当前基线，不再继续围绕其做连续微调。
- 核心经验：对 guard 的补强更应优先“成型稳定性”，而非单纯抬高早期强度。

## 三、后续 Act2/Act3 可直接继承的工作规则

### 1) 调参方法论（建议强制遵守）

- 只做单变量小步实验（一次只改一个参数或一个点位）。
- 每轮固定 runs/seed 对比，避免口径漂移。
- 同步输出按怪拆分（attempts/deaths/winRate），避免被总通过率误导。
- 先验证“主要压力来源是基础输出还是惩罚链”，再决定改哪一层。

### 2) 变更边界（建议沿用）

- 优先不改 policy/map/Boss，先在最小局部验证。
- 不做“全体统一降难度”；保持筛选身份。
- 当某路线表现异常时，先补“成型桥梁能力”，再动敌人基础数值。

### 3) 观测指标优先级（Act2/Act3）

第一层（必看）：

- `firstEliteWin`（按路线）
- `firstEliteAttemptsByMonsterId`
- `firstEliteDeathsByMonsterId`
- `firstEliteWinRateByMonsterId`

第二层（风险信号）：

- `mixed` 样本规模变化
- `non-battle` 占比变化
- 路线选择倾向是否出现异常偏移

第三层（诊断补充）：

- 关键失败前 2-3 回合摘要
- 到达关键战前 deck 成型指标（桥梁/防御核心/资源转换牌）

## 四、给 Act2/Act3 的落地建议（执行顺序）

1. 先复制 Act1 的多 seed 验证流程（至少 5 seed，runs=50）做“纯观察首轮”。
2. 建立 Act2/Act3 对应的“按怪拆分”输出，再开始平衡改动。
3. 若出现单路线 0% 或接近 0%，优先补构筑桥梁，不直接整体下调怪物。
4. 若某怪在多路线均形成主杀手，再做单变量怪物回调。
