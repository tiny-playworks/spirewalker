# Spirewalker v0.5 计划：把行者做成第一个正式角色，并让两条构筑真正成立

## Summary

`v0.5` 继续坚持单角色深化，不做三角色，不开第二资源系统。  
版本目标收紧为 4 件事：

- 行者 starter 首回合稳定触发 `momentum`
- 行者至少形成 **2 条真构筑分支**
- 每条分支都绑定 **2 张核心牌 + 1 个遗物**
- 奖励池改为 **角色主导 + 少量通用卡混入**，而不是 100% 角色化

`v0.5` 的关键不是“再加内容”，而是让行者 run 从头到尾都像行者 run。

## Key Changes

### 1. 行者 starter 改成“首回合稳定起势”

角色仍只做 `行者`，但 starter 必须提高核心牌密度。  
starter 固定为 10 张：

- `3 strike`
- `3 defend`
- `2 prime_rhythm`
- `1 brace_rhythm`
- `1 steady_step`

配合现有角色被动“战斗开始时获得 1 层连势”，这样首回合有 `3/10` 的起势入口牌，5 张起手时未摸到核心牌的概率约为 `8.3%`，足够满足 `v0.5` 的“稳定触发核心机制”要求。

角色定义需要正式接管开局数据，`CharacterDefinition` 至少包含：

- `baseMaxHp`
- `starterDeck`
- `startingRelics`
- `startingPotions`
- `rewardCardPool`
- `rewardRelicPool`
- `passive`

`createMapRun` 不再硬编码开局生命、卡组、药水，而是统一读取角色定义。

### 2. 两条构筑分支必须“成套成立”

`v0.5` 只承认两条分支，且每条分支必须同时满足：

- `2 张核心牌`
- `1 个核心遗物`

否则不计入“构筑成立”。

两条分支固定为：

- **保势防守线**
  - 核心牌：`brace_rhythm`、`tempo_guard`
  - 核心遗物：`guard_knot`
  - 目标体验：稳定保留连势，靠防守与低风险兑现滚优势
- **快速兑现线**
  - 核心牌：`burst_strike`、`snap_strike`
  - 核心遗物：`burst_emblem`
  - 目标体验：快速起势后尽快把连势换成伤害，强调爆发节奏

`v0.5` 新增内容要围绕这两条分支服务，不再发明第三条路线。

新增内容量固定为：

- 新增卡牌 6 张
  - `2 张保势线补强`
  - `2 张兑现线补强`
  - `2 张通用节奏修复牌`
- 新增遗物 4 个
  - `2 个偏保势`
  - `2 个偏兑现`

### 3. 奖励池改成 75% 角色卡 + 25% 通用卡

奖励池不做 100% 角色化，固定采用：

- `75%` 当前角色卡池
- `25%` 通用卡池

通用卡池限定为当前已有的通用基础牌和低复杂度功能牌，不放与行者主轴冲突的新机制。

实现上不做复杂加权系统，只做最小分发规则：

- 每次生成 `card_choice`
- 先按张位独立抽取来源：
  - `75%` 概率从 `character.rewardCardPool`
  - `25%` 概率从 `COMMON_REWARD_CARD_POOL`
- 再做去重与合法性处理

遗物池仍按角色优先，但不要求混入通用遗物比例；Boss / 精英先抽角色池，池空再回落公共池。

### 4. 敌人反制只用两种明确触发模板

为了方便实现，`v0.5` 的反制敌人只允许用两种明确触发条件：

- **多打惩罚型**
  - 触发条件：`playerCardsPlayedThisTurn >= 3`
  - 结果：敌人获得固定格挡
  - 继续沿用现有 `punish_multi_play`
- **削势型**
  - 触发条件：敌人该轮执行“削减状态意图”时
  - 结果：固定削减 `momentum`
  - 继续沿用现有 `reduce_status`

不在 `v0.5` 引入更复杂的“看玩家当前连势层数再分支”的 AI 框架。  
只需要把触发条件在文案和定义里写死：

- `玩家本回合出牌 >= 3 时，获得 8 格挡`
- `本次行动削减你 2 层连势`

## Implementation Changes

### 角色与开局

- 扩展 `CharacterDefinition`，让角色成为开局和成长的唯一来源
- `starterDeck.ts` 退出“全局唯一 starter”角色，starter 由角色定义提供
- 当前入口仍默认 `walker`，不做角色选择 UI

### 内容包组织

- 保留当前 `momentum` 作为唯一核心差异化机制
- `v0.5` 新卡 / 新遗物 / 新事件都围绕“保势线 / 兑现线”组织
- 不改现有 `EffectDefinition + custom script + statusRegistry` 结构
- 不引入 Guard / Focus / Combo 这类新资源名词

### 奖励与事件

- 新增一个 `COMMON_REWARD_CARD_POOL`
- 战后奖励按 `75/25` 比例混抽角色卡与通用卡
- 新增 2 个事件，且都直接服务行者路线：
  - 1 个偏保势
  - 1 个偏兑现
- 事件结果仅使用现有原语：
  - 加血 / 扣血 / 金币 / 遗物 / 药水 / 给牌 / 删牌

## Test Plan

必须补的测试：

- `createMapRun` 会从角色定义读取 `HP / starterDeck / potion / relic`
- 行者 starter 固定为 `3 strike + 3 defend + 2 prime_rhythm + 1 brace_rhythm + 1 steady_step`
- 首回合抽 5 张时，行者 starter 能稳定摸到起势牌
  - 测试口径：固定种子集下首战起手至少有 1 张起势牌的比例满足预期
- 卡牌奖励按 `75% 角色池 + 25% 通用池` 分发
- 两条构筑都满足“2 张核心牌 + 1 个遗物”
- 多打惩罚敌人的触发条件固定为 `playerCardsPlayedThisTurn >= 3`
- 削势敌人的触发条件固定为“轮到该意图时削减固定层数”
- 至少 1 组集成测试验证：
  - 保势线能在普通战中稳定保留 momentum
  - 兑现线能在普通战中明显打出高于基础攻击的爆发

人工验收标准：

- 第一场战斗就能稳定看到 `momentum`
- 普通一局中能清楚分辨自己更偏“保势”还是“兑现”
- 奖励页里不再全是行者牌，也不会被通用牌冲淡主线
- 敌人反制是“规则清楚”，不是“我不知道它为什么突然克我”

## Assumptions

- `v0.5` 只做 `行者`
- 不做角色选择界面
- 不新增第二资源系统
- 敌人反制继续建立在现有 `reduce_status / punish_multi_play` 上，不扩 AI 框架
- 地图长度、经济重做、事件大扩容不作为 `v0.5` 主目标
